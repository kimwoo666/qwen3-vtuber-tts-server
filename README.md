# qwen3-vtuber-tts-server

OpenAI-compatible Qwen3 TTS server and prompt generator for local voice-clone workflows.

This repository is designed to pair well with projects such as Open-LLM-VTuber, but it is not tied to any one frontend. It exposes a small FastAPI server that accepts `POST /v1/audio/speech` requests and returns WAV audio generated by local Qwen3-TTS models.

This is an unofficial wrapper. It is not affiliated with, endorsed by, or maintained by Qwen, Alibaba Cloud, OpenAI, GitHub, or Open-LLM-VTuber.

## What is included

- `qwen3-vtuber-tts`: FastAPI server entry point
- `qwen3-vtuber-tts-prompt`: CLI for generating reusable voice-clone prompt assets
- Testable helper modules for settings, request parsing, and text preprocessing

## What is intentionally not included

- Qwen3 model weights
- Generated prompt assets
- Personal reference audio
- Hard-coded voice personas or private voice settings

If you publish this repository, keep your own reference audio and generated prompt files outside Git.

## Responsible use

- Only use reference audio and prompt assets that you created yourself or that you have explicit permission to use.
- Do not use this project for impersonation, fraud, harassment, or deceptive synthetic media.
- Do not publish private voice samples, generated prompt assets, API keys, or personal local paths to GitHub.

## Requirements

- Python 3.10+
- A clean virtual environment is strongly recommended
- Local access to the Qwen3-TTS models you want to use

This project installs the [`qwen-tts`](https://pypi.org/project/qwen-tts/) package, but you still need to download the model weights separately. See the upstream Qwen3-TTS project for the latest installation guidance:

- [Qwen3-TTS GitHub repository](https://github.com/QwenLM/Qwen3-TTS)
- [Qwen3-TTS PyPI package](https://pypi.org/project/qwen-tts/)

## Quick start

### 1. Create a fresh environment

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -e .
```

### 2. Download the model weights

You can keep the models anywhere on disk. A common local layout is:

```text
D:\models\qwen3-tts\
  Qwen3-TTS-12Hz-1.7B-Base\
  Qwen3-TTS-12Hz-1.7B-VoiceDesign\
  voice_assets\
```

The server uses:

- `Qwen3-TTS-12Hz-1.7B-Base` for inference
- `Qwen3-TTS-12Hz-1.7B-VoiceDesign` when creating a reusable prompt

### 3. Generate a reusable prompt asset

You must supply the prompt design text yourself. This repository does not ship a built-in voice persona.

```powershell
$env:QWEN3_TTS_ROOT_DIR = "D:\models\qwen3-tts"
$env:QWEN3_TTS_ASSET_DIR = "D:\models\qwen3-tts\voice_assets"

qwen3-vtuber-tts-prompt `
  --voice-name default `
  --language English `
  --reference-text "Take a breath. We will solve this one step at a time." `
  --reference-instruction "Young adult voice. Calm, friendly, clear diction, steady pacing."
```

This writes:

- `voice_assets/default_reference.wav`
- `voice_assets/default_prompt.pkl`
- `voice_assets/default_metadata.json`

You can also provide the same values with environment variables:

- `QWEN3_TTS_REFERENCE_TEXT`
- `QWEN3_TTS_REFERENCE_INSTRUCT`

### 4. Start the server

```powershell
$env:QWEN3_TTS_ROOT_DIR = "D:\models\qwen3-tts"
$env:QWEN3_TTS_ASSET_DIR = "D:\models\qwen3-tts\voice_assets"

qwen3-vtuber-tts --host 127.0.0.1 --port 8000
```

The server exposes:

- `GET /`
- `GET /health`
- `POST /v1/audio/speech`

## API behavior

The server accepts OpenAI-style speech requests and returns `audio/wav`.

Example request:

```powershell
$body = @{
  model = "qwen3-tts-en-single"
  voice = "default"
  input = "Hello from a local Qwen3 server."
  response_format = "wav"
  language = "English"
} | ConvertTo-Json

Invoke-WebRequest `
  -Uri "http://127.0.0.1:8000/v1/audio/speech" `
  -Method Post `
  -ContentType "application/json" `
  -Body $body `
  -OutFile ".\sample.wav"
```

Notes:

- Only `wav` output is supported.
- Known style prefixes such as `[warm]` are stripped before synthesis.
- The `voice` value selects `<voice>_prompt.pkl` in the asset directory. If that file does not exist, the server falls back to `default_prompt.pkl`.

## Troubleshooting

### `Missing runtime dependency`

Reinstall into a clean environment:

```powershell
python -m pip install --upgrade pip
python -m pip install -e .
```

### `Missing prompt file`

Generate a prompt asset first, or point `QWEN3_TTS_ASSET_DIR` at the directory that already contains your `<voice>_prompt.pkl` files.

### CUDA or memory issues

- Try `QWEN3_TTS_DEVICE=cpu` for a functional fallback
- Try `QWEN3_TTS_DTYPE=float32` if half precision is not stable on your setup
- Disable warmup with `qwen3-vtuber-tts --skip-warmup` when debugging startup

## Privacy note

Reference audio and generated prompt assets may contain personally identifiable voice information. Treat them as private biometric data and avoid committing them to GitHub.
